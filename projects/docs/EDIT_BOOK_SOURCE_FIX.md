# edit_book_source 工具死循环修复报告

> **修复时间**: 2025-01-20
> **修复文件**: `src/tools/book_source_editor.py`
> **问题类型**: 死循环/重复调用
> **严重程度**: ⭐⭐⭐⭐⭐（高）

---

## 📋 问题描述

### 问题表现

智能体在调用 `edit_book_source` 工具创建书源后，可能继续调用该工具或其他工具来"完善"书源，导致：
- 重复调用 `edit_book_source`
- 无限循环
- 资源浪费
- 书源无法完成

### 问题原因

1. **工具返回提示不够强烈**
   - 原来的提示虽然要求"不要调用任何工具"
   - 但不够明确和强烈
   - 智能体可能忽略这个提示

2. **缺少状态追踪**
   - 没有机制追踪工具是否已被调用
   - 无法阻止重复调用

3. **智能体行为模式**
   - 智能体可能在检查返回内容后，认为需要"改进"
   - 尝试再次调用工具修改字段
   - 循环往复

---

## 🛠️ 修复方案

### 方案1：添加全局状态追踪（核心修复）

**实现**：
```python
# 全局状态追踪：防止重复调用edit_book_source
_edit_book_source_called = False
_last_call_time = None

@tool
def edit_book_source(...):
    global _edit_book_source_called, _last_call_time

    # 🚨 防止重复调用（死循环保护）
    if _edit_book_source_called:
        return """
## 🚨 🚨 🚨 严重错误：重复调用 edit_book_source！🚨 🚨 🚨

**错误原因**：edit_book_source 工具已经被调用过了！

**禁止行为**：
- ❌ edit_book_source 只能被调用一次
- ❌ 不能再次调用此工具修改字段
- ❌ 不能调用任何其他工具

**正确做法**：
- ✅ 直接将之前生成的书源JSON复制给用户
- ✅ 用文字回复用户的问题（不要调用工具）
- ✅ 告诉用户书源已完成

**书源已完成，请直接输出给用户！不要调用任何工具！**

---

【系统指令】：立即停止所有工具调用，直接输出之前的书源JSON！
"""

    # 标记工具已被调用
    _edit_book_source_called = True
    _last_call_time = datetime.now()
```

**优点**：
- ✅ 100%防止重复调用
- ✅ 简单有效
- ✅ 不影响正常使用

1. **双重保护**

   - 禁止的行为
   - 必须执行的操作

- 只能在第三阶段调用 `edit_book_source` 一次
- 调用后必须立即停止所有工具调用
- 直接输出JSON给用户
- 用文字回复用户的后续问题

**对于用户**：
- 接收完整的书源JSON
- 直接导入Legado APP
