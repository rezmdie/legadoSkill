# 编码检测功能说明

## 概述

系统已实现智能编码检测功能，支持自动检测和手动指定网页编码，解决了 POST 请求中文乱码等编码问题。

## 功能特性

### 1. 智能编码检测

系统按以下优先级自动检测网页编码：

1. **用户指定的 charset 参数**（最高优先级）
2. **HTTP Content-Type header 中的 charset**
3. **HTML meta 标签中的 charset**
4. **chardet 库智能检测**（置信度 > 0.7）
5. **默认 utf-8**（兜底方案）

### 2. 支持的编码

- UTF-8 / UTF8
- GBK / GB2312 / GB18030
- 其他标准编码

### 3. 编码标准化

系统会自动标准化编码名称：
- `gbk`, `gb2312`, `gb18030` → `gbk`
- `utf-8`, `utf8` → `utf-8`

## 使用方法

### 方法 1：使用 smart_fetch_html 工具

```python
# 自动检测编码
smart_fetch_html(
    url="https://example.com/search",
    method="POST",
    data='{"keyword": "斗破苍穹"}'
)

# 指定 GBK 编码
smart_fetch_html(
    url="https://example.com/search",
    method="POST",
    data='{"keyword": "斗破苍穹"}',
    charset="gbk"
)

# 指定 UTF-8 编码
smart_fetch_html(
    url="https://example.com/search",
    method="GET",
    charset="utf-8"
)
```

### 方法 2：直接使用 SmartRequest

```python
from src.utils.smart_request import get_smart_request

requester = get_smart_request()

# 自动检测编码
result = requester.fetch(
    url="https://example.com",
    method="GET"
)

# 指定编码
result = requester.fetch(
    url="https://example.com",
    method="POST",
    data={"keyword": "关键词"},
    charset="gbk"
)

print(f"检测到的编码: {result['encoding']}")
print(f"HTML内容: {result['html']}")
```

## 书源开发中的应用

在 Legado 书源开发中，对于使用 GBK 编码的网站，可以这样配置：

```json
{
  "searchUrl": "https://example.com/search,{\"charset\": \"gbk\", \"method\": \"POST\", \"body\": \"keyword={{key}}&page={{page}}\"}",
  "ruleSearch": {
    "bookList": ".book-item",
    "name": ".title",
    "author": ".author"
  }
}
```

系统会自动：
1. 识别 JSON 配置中的 `charset` 字段
2. 使用指定的编码发送 POST 请求
3. 正确解码响应内容

## 技术实现

### 核心方法：`_detect_encoding()`

位置：`src/utils/smart_request.py`

```python
def _detect_encoding(self, response: requests.Response, charset: Optional[str] = None) -> str:
    """
    智能检测响应编码
    
    优先级：
    1. 用户指定的 charset 参数
    2. HTTP Content-Type header 中的 charset
    3. HTML meta 标签中的 charset
    4. chardet 库检测
    5. 默认 utf-8
    """
```

### 关键改进

1. **不再依赖 response.text**
   - 旧方法：使用 `response.text`（编码可能不准确）
   - 新方法：使用 `response.content.decode(detected_encoding)`

2. **多层检测机制**
   - 从 HTTP header 提取编码
   - 从 HTML meta 标签提取编码
   - 使用 chardet 库智能检测

3. **容错处理**
   - 解码失败时自动回退到 utf-8
   - 使用 `errors='replace'` 避免解码异常

## 注意事项

1. **charset 参数是可选的**
   - 不指定时，系统会自动检测
   - 建议对已知编码的网站明确指定

2. **GBK 网站特别注意**
   - 很多中文小说网站使用 GBK 编码
   - 必须在书源配置中指定 `"charset": "gbk"`

3. **编码检测的局限性**
   - chardet 检测不是 100% 准确
   - 对于已知编码的网站，建议手动指定
