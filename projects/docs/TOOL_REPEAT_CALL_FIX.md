# 工具重复调用问题修复说明

## 📋 问题描述

当用户问"xxxxx这个网站的搜索规则该怎么写"这类问题时，Agent会陷入循环重复调用工具的问题，导致无法给出正确的答案。

### 具体表现
- 反复调用 `search_knowledge` 工具
- 不知道应该先访问网站获取源代码
- 无法给出准确的选择器规则

## 🔍 问题根源

经过分析，发现问题出在以下几个方面：

### 1. 缺少网站访问工具
- 工具列表中没有 `fetch_web_page` 等访问网站的工具
- Agent无法获取网站的实际HTML源代码
- 只能根据猜测编写规则

### 2. System Prompt流程不明确
- System Prompt中没有明确说明"网站规则编写"的标准流程
- Agent不知道应该先访问网站
- 缺少禁止重复调用的约束

### 3. 工具调用优先级不清晰
- 知识库搜索优先级过高
- 缺少针对特定场景的调用策略

## ✅ 解决方案

### 1. 新增网站访问工具

创建了 `src/tools/web_fetch_tool.py`，包含以下工具：

#### fetch_web_page ⭐⭐⭐⭐⭐
**功能**：获取网站的HTML源代码

**参数**：
- `url`: 网站URL
- `render_js`: 是否渲染JavaScript（默认False）
- `timeout`: 请求超时时间（默认30秒）

**返回**：
- HTML源代码
- 页面结构分析
- 关键HTML片段
- 下一步操作建议

**示例**：
```
fetch_web_page(url="https://example.com")
```

#### extract_elements ⭐⭐⭐⭐
**功能**：从HTML中提取特定元素

**参数**：
- `html`: HTML源代码
- `selector`: CSS选择器
- `tag`: HTML标签名
- `class_name`: class属性值
- `id_name`: id属性值

**返回**：
- 匹配的元素列表
- 元素详细信息
- 推荐的选择器

#### analyze_search_structure ⭐⭐⭐
**功能**：分析网站的搜索页面结构

**参数**：
- `url`: 搜索页面的URL

**返回**：
- 搜索表单分析
- 搜索容器识别
- 书源编写建议

### 2. 更新工具列表

在 `config/agent_llm_config.json` 中更新工具列表，将新工具放在第一位：

```js
{
  "tools": [
    "fetch_web_page",          // 新增：网站访问（最高优先级）
    "extract_elements",        // 新增：元素提取
    "analyze_search_structure", // 新增：搜索结构分析
    "search_knowledge",        // 知识库搜索
    ...
  ]
}
```

### 3. 更新System Prompt

在 `config/agent_llm_config.json` 中新增"网站规则编写流程"部分：

#### 标准流程

当用户问"xxxxx这个网站的搜索规则该怎么写"时，必须严格按照以下流程：

**第1步：获取网站源代码（必须）**
```
使用 fetch_web_page 工具获取目标网站的HTML源代码
```

**第2步：分析页面结构（必须）**
```
使用 fetch_web_page 工具的返回结果，查看"页面结构分析"部分
```

**第3步：搜索知识库（可选但推荐）**
```
使用 search_knowledge 工具搜索相关的规则说明
```

**第4步：编写书源规则（核心）**
```
使用 edit_book_source 工具创建或编辑书源
```

**第5步：测试验证（必须）**
```
建议用户在Legado APP中实际测试
```

#### 禁止行为

绝对禁止以下行为：
1. ❌ **跳过网站访问**：直接根据猜测编写规则
2. ❌ **重复调用工具**：同一个网站重复访问多次
3. ❌ **无限循环调用**：反复调用 search_knowledge 或其他工具
4. ❌ **编造规则**：不查看页面结构就编写选择器
5. ❌ **忽略知识库**：不搜索知识库就直接回答

#### 正确示例

**用户问**："起点中文网的搜索规则该怎么写？"

**正确的流程**：
1. 调用 `fetch_web_page` 访问起点中文网的搜索页面
2. 查看返回的页面结构分析
3. 调用 `search_knowledge` 搜索"搜索规则"相关文档
4. 调用 `edit_book_source` 创建书源，填入正确的选择器
5. 建议用户在APP中测试

**错误的流程**：
1. ❌ 直接调用 `search_knowledge` 搜索"起点中文网"
2. ❌ 反复调用 `search_knowledge` 搜索不同的关键词
3. ❌ 不访问网站就直接给出选择器

## 📝 注意事项

1. **工具依赖**：新工具需要安装 `requests` 和 `beautifulsoup4` 库
2. **超时设置**：某些网站可能需要增加超时时间
3. **JS渲染**：动态网站可能需要使用 `render_js=True`（需要额外配置）
4. **反爬机制**：某些网站可能有反爬机制，需要特殊处理
